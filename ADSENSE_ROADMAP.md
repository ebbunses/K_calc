# k-calc-project AdSense 수익화 로드맵

작성일: 2026-05-23
대상 사이트: 한국형 계산기 모음 (대출 / 연봉 / 적금 / 퇴직금)

현 상태: AdSense 미신청, 배포 환경 미정

## 0. 전체 흐름 한눈에 보기

```

[2단계] 도메인 확보 & 배포          ├─► [4단계] AdSense 신청 (심사 1~14일)

                                          [5단계] 광고 코드 통합·게재
```

각 단계는 의존성이 있어 순서를 건너뛰면 심사 거절 또는 수익 손실이 발생합니다. 특히 1~3단계를 충분히 다지지 않고 4단계로 가면 "가치가 낮은 콘텐츠" 사유로 거절되는 경우가 많습니다.

## 1단계 — 콘텐츠·필수 페이지 정비 (예상 1~2주)

### 1.1 AdSense가 요구하는 필수 페이지
한국 사이트가 가장 자주 거절되는 항목입니다. 아래 페이지는 *반드시* 사이트 푸터에서 접근 가능해야 합니다.

| 페이지 | 경로 예시 | 핵심 포함 사항 |
|---|---|---|
| 개인정보처리방침 | `/privacy` | 수집 항목 없음 명시, **AdSense 쿠키 사용 고지**, Google 광고 정책 링크, 제3자 광고사업자(Google) 사용 고지 |
| 이용약관 | `/terms` | 계산 결과의 참고용 한계, 면책 조항 |
| 사이트 소개 | `/about` | 운영자(개인/팀), 운영 목적, 업데이트 정책 |
| 문의 | `/contact` | 이메일 or 폼. 폼이면 reCAPTCHA 권장 |
| 면책 고지 | 계산기 페이지 하단 | "본 계산 결과는 법적 효력이 없으며 참고용입니다" |

> 한국 AdSense 심사에서 가장 많은 거절 사유: ① 콘텐츠 부족, ② 개인정보처리방침 누락 또는 부실, ③ 면책 고지 없음(특히 금융 계산기는 필수). 퇴직금·대출 계산기는 "민감한 금융 정보"로 분류될 수 있어 면책이 더욱 중요합니다.

### 1.2 콘텐츠 깊이 보강
계산기 단일 페이지 4개만으로는 심사 통과가 어렵습니다. 각 계산기 페이지에 **설명 콘텐츠 800~1500자**를 추가하세요.

각 계산기 페이지에 넣을 콘텐츠 구조 (예: `/severance`):
1. 퇴직금이란? — 정의, 근로기준법 근거 (근거 법령 명시)
2. 계산 공식 — 통상임금/평균임금 차이, 계산식 예시
3. 자주 묻는 질문 5~10개 (FAQ JSON-LD 적용 가능)
4. 관련 정보 — 퇴직소득세, 중간정산 요건 등

이렇게 하면 페이지 체류시간이 늘어나고(광고 노출 ↑), 롱테일 검색 키워드 진입(예: "1년 미만 퇴직금")이 가능해집니다.

### 1.3 콘텐츠 추가 권장
신청 시점 기준 최소 페이지 수 목표:
- 계산기 4개 (현재 보유)
- 가이드/블로그 글 6~10개 (예: "2026년 연봉 실수령액 변화", "주담대 금리 인상기 대출 전략")
- 필수 페이지 4개 (위 표)

→ **총 14~18페이지** 정도가 안전한 심사 통과 라인입니다.

## 2단계 — 도메인 확보 & 배포 (예상 1~3일)

### 2.1 도메인
- **루트 도메인**(`example.com`)으로 신청해야 합니다. 무료 서브도메인(.vercel.app, .netlify.app)은 AdSense 심사 거절됩니다.
- `.com`, `.co.kr`, `.kr` 모두 가능. 가격·갱신 정책상 `.com` 권장.
- 도메인 등록 후 **최소 1~2주 운영 후** 신청하는 것이 통과율이 높습니다.

### 2.2 호스팅
배포 환경은 미정이라 하셨지만, 이 프로젝트(Next.js 16 + App Router)에 가장 적합한 옵션:

| 옵션 | 장점 | 비고 |
|---|---|---|
| Vercel | Next.js 네이티브, 무료 플랜 충분, ENV 관리 쉬움 | 한국 트래픽이 많으면 Edge가 도쿄 리전 사용 |
| Cloudflare Pages | 무료 플랜 트래픽 무제한, CDN 빠름 | Next.js 16 App Router 호환 검증 필요 |
| 자체 서버 / NAS | 완전한 통제 | SSL/HTTPS 직접 관리, AdSense는 HTTPS 필수 |

→ 1차 권장: **Vercel + 커스텀 도메인**. 추후 트래픽이 일 1만 PV를 넘으면 Cloudflare 전환 고려.

### 2.3 배포 전 체크
- HTTPS (필수)
- 404 페이지 동작
- robots.txt 허용
- 모바일 반응형 (현재 Tailwind로 잘 되어 있을 가능성 높음)

## 3단계 — SEO·트래픽 기반 다지기 (지속 작업, 신청 전 최소 2주 운영)

### 3.1 키워드 우선순위 (한국 검색량 추정)
| 계산기 | 핵심 키워드 | 월간 검색량(대략) | 경쟁도 |
|---|---|---|---|
| 연봉 | "연봉계산기", "실수령액 계산기" | 매우 높음 | 매우 높음 |
| 퇴직금 | "퇴직금계산기", "퇴직금 계산" | 높음 | 높음 |
| 적금 | "적금계산기", "예적금 이자 계산" | 중간 | 중간 |
| 대출 | "대출이자계산기", "원리금균등상환" | 높음 | 높음 |

**전략**: 메인 키워드는 경쟁이 치열하므로, **롱테일**(예: "1년차 신입 연봉 실수령액", "DSR 계산기", "퇴직금 1년 미만") 콘텐츠를 보조 페이지로 추가해 유입 확보.

### 3.2 기술 SEO 체크리스트
- [ ] `app/layout.js`에 `metadata` 객체로 기본 title/description/openGraph 설정
- [ ] 각 페이지 `generateMetadata`로 페이지별 메타 차등 적용
- [ ] `app/sitemap.js` 또는 `public/sitemap.xml` 자동 생성
- [ ] `public/robots.txt` (Allow: /, Sitemap: 링크 포함)
- [ ] JSON-LD 구조화 데이터: `Calculator`/`WebApplication`/`FAQPage` 스키마
- [ ] Open Graph 이미지 (1200×630)
- [ ] 파비콘, manifest, 모바일 테마 컬러
- [ ] Core Web Vitals: LCP < 2.5s, CLS < 0.1, INP < 200ms (광고 게재 후 CLS 특히 주의)

### 3.3 검색엔진 등록
| 엔진 | 도구 | 비고 |
|---|---|---|
| Google | Search Console | 사이트맵 제출, 색인 요청 |
| Naver | Search Advisor | 한국 트래픽의 30~40%, 필수 |
| Bing | Webmaster Tools | 옵션, 소요 적음 |
| Daum | (Naver와 별개로 등록) | 비중 낮음 |

### 3.4 분석 도구
- **Google Analytics 4** — 광고 수익과 연관 분석
- **Microsoft Clarity** — 무료 히트맵·세션 녹화. 광고 위치 최적화에 매우 유용
- Search Console과 GA4 연동

### 3.5 최소 트래픽 가이드라인
AdSense는 공식적인 최소 트래픽 기준이 없지만, 한국 심사 통과 경험상 안전선:
- **일 방문자 50~100명** 이상
- **신청 시점 사이트 운영 기간 2~4주** 이상
- 자연 검색 유입이 직접 유입보다 많을 것

## 4단계 — AdSense 신청 (예상 1~14일 심사)

### 4.1 신청 전 최종 점검
- 필수 페이지 4종 푸터 링크 ✓
- 각 계산기 페이지 설명 콘텐츠 충분 ✓
- 모든 페이지 모바일 정상 동작 ✓
- 404·500 페이지 정상 ✓
- 사이트맵·robots.txt 정상 ✓
- HTTPS ✓

### 4.2 신청 절차
1. https://www.google.com/adsense 접속 → Google 계정으로 가입
2. 사이트 URL 입력 (루트 도메인)
3. 국가: 대한민국 / 결제 통화: KRW or USD
4. **고유 코드 스니펫**을 사이트 `<head>`에 삽입 → 이 단계만 사전에 코드를 넣어야 합니다 (5단계와 연결)
5. 심사 대기 (보통 1~14일, 길게는 한 달)

### 4.3 신청 시 자주 묻는 사항
- **개인 vs 사업자**: 개인으로 신청 가능. 사업자 등록 없어도 됨. 다만 연 600만 원 이상 수익 시 종합소득세 신고 의무.
- **주소·이름**: PIN 우편 발송 시 정확해야 함 (10$ 이상 누적 시 발송).
- **수취 계좌**: 외화통장(KEB하나/우리 등) 또는 일반 원화통장 가능. SWIFT 코드 필요.

### 4.4 거절 시 대응
가장 흔한 거절 사유와 즉시 처치:
| 사유 | 처치 |
|---|---|
| 가치가 낮은 콘텐츠 | 페이지당 글자 수·이미지·내부 링크 보강 |
| 콘텐츠 정책 위반 | 면책 고지 추가, 금융 조언으로 오인될 표현 수정 |
| 사이트 탐색 어려움 | 헤더 메뉴, 빵부스러기, 사이트맵 추가 |
| 사용 불가 사이트 | 도메인 DNS·SSL·접속 안정성 점검 |

거절 후 수정 → 재신청은 즉시 가능. 같은 실수 반복하면 계정이 영구 정지될 수 있으니 거절 사유를 정확히 분석.

## 5단계 — 광고 코드 통합·게재 (예상 0.5~1일 작업)

승인 후 또는 심사 직전 사이트에 코드를 넣어야 합니다. Next.js 16 App Router 기준 권장 구조:

### 5.1 파일 구조 (계획)
```
app/
  layout.js                    # AdSense 스크립트 (next/script)
  components/
    ads/
      AdSlot.jsx              # 광고 단위 컴포넌트
      AutoAd.jsx              # 자동 광고 토글 (옵션)
public/
  ads.txt                     # google.com, pub-XXX, DIRECT, ...
.env.local
  NEXT_PUBLIC_ADSENSE_CLIENT=ca-pub-XXXXXXXXXXXXXXXX
```

### 5.2 통합 방식 선택지
| 방식 | 장점 | 단점 | 권장도 |
|---|---|---|---|
| 자동 광고만 | 설치 5분, Google이 알아서 배치 | 위치 제어 불가, CLS 위험, 결과 카드 위에 광고 끼어들기 | ★★ |
| 수동 광고 단위만 | 위치 완전 제어, CLS 통제 | 위치 실험에 시간 듦 | ★★★ |
| 자동 + 수동 혼합 | 핵심 위치는 수동, 나머지는 자동 | 설정 복잡 | ★★★★ (권장) |

### 5.3 계산기 페이지 권장 배치 (수익률 기준)
```
┌──────────────────────────┐
│        헤더               │
├──────────────────────────┤
│   [광고 A] 디스플레이      │  ← 첫 화면 노출, 직사각형
├──────────────────────────┤
│   계산기 입력 폼           │
│                          │
│   [결과 카드]             │
├──────────────────────────┤
│   [광고 B] 인피드/네이티브 │  ← 결과 직후, 가장 높은 CTR
├──────────────────────────┤
│   계산 공식 설명           │
│   FAQ                    │
├──────────────────────────┤
│   [광고 C] 디스플레이      │
├──────────────────────────┤
│   푸터                    │
└──────────────────────────┘
모바일: 추가로 sticky bottom 광고 1개 (수익률 매우 높음)
```

**중요**: 결과 카드 *위*에 광고를 넣지 마세요. 사용자가 계산 결과를 보러 왔는데 광고가 가리면 직행률(즉시 이탈)이 급등합니다.

### 5.4 CLS 방지 원칙
- 모든 광고 슬롯은 `min-height` 고정 (예: 250px)
- `data-full-width-responsive="true"` 사용 시 부모 컨테이너 max-width 명시
- 모바일에서 광고 사이즈가 동적 변경되지 않도록 `data-ad-format="auto"`만 쓰지 말고 명시 사이즈 병행

### 5.5 ads.txt
승인 후 발급되는 퍼블리셔 ID로 `public/ads.txt` 생성:
```
google.com, pub-XXXXXXXXXXXXXXXX, DIRECT, f08c47fec0942fa0
```
이 파일이 없으면 일부 입찰자가 광고를 안 띄워 수익 손실.

## 6단계 — 배치 최적화·수익 모니터링 (지속)

### 6.1 첫 2주: 데이터 수집
- AdSense 보고서: 페이지 RPM, CTR, 노출
- GA4와 연동: 페이지별 수익, 사용자 흐름
- Clarity: 광고 영역 클릭 패턴

### 6.2 최적화 레버 (효과 순)
1. **모바일 sticky 광고 활성화** — 모바일 수익 +30~50%
2. **결과 카드 직후 인피드 광고** — CTR 가장 높음
3. **광고 사이즈 실험** — Large Rectangle (336×280)이 일반적으로 가장 높음
4. **자동 광고 강도 조절** — 너무 강하면 UX 손상, 너무 약하면 수익↓. 50~70% 권장
5. **페이지 속도** — LCP 1초 줄이면 광고 노출 +10~15%

### 6.3 피해야 할 것
- 본인이 광고 클릭 (즉시 영구 정지)
- "광고를 클릭해주세요" 같은 유도 문구 (정책 위반)
- 콘텐츠 없이 광고만 가득한 페이지
- 팝업/이미지 위에 광고 겹침

## 7단계 — 정책·세무·확장

### 7.1 한국 세무
- **연 수익 600만 원 미만**: 종합소득세 신고 시 기타소득 합산.
- **600만 원 이상**: 사업자 등록(통신판매업 또는 1인 미디어 콘텐츠 창작자) 권장. 부가세 신고 면제 가능 (수출용역).
- AdSense는 **외화 수익**이므로 부가세 영세율 적용 가능 → 세무 절감 효과 큼.
- **Google에 W-8BEN 제출** — 미국 원천세 면제 (한미 조세조약).

### 7.2 정책 모니터링
- 매월 AdSense 정책 센터 확인
- 무효 클릭 보고서 주시 (CTR이 5% 이상이면 비정상으로 의심됨, 일반적으로 1~3%)
- 사용자 약관 변경 시 개인정보처리방침 동기화

### 7.3 수익 확장 (광고 외)
계산기 사이트는 광고 단독 수익이 한계가 있습니다. 트래픽이 일 5000 PV를 넘으면 다음을 검토:
- **제휴 마케팅** — 대출(토스/카카오뱅크), 적금(저축은행), 보험 비교 링크. 광고 RPM 대비 5~20배.
- **유료 PDF 리포트** — 예: 퇴직금 절세 가이드 PDF
- **API 판매** — 계산 로직을 다른 사이트에 API로 제공 (B2B)
